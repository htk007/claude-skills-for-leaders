## The Problem

Most Claude skills are built for developers writing code. But **leaders** spend their days on a completely different kind of work — performance reviews, architectural decisions, stakeholder updates, sprint planning, incident post-mortems, and promotion cases.

Writing a performance review from scratch takes hours. A code review comment with the wrong tone starts a conflict. A hastily written job description attracts the wrong candidates. A vague ADR gets ignored six months later.

Claude can help with all of this — but only with the right skill behind it.

**This repo is that skill library.**

---

## 🚀 Quick Start

```
1. Open Claude.ai → Settings → Claude Code Skills
2. Click "New Skill"
3. Copy any SKILL.md file from this repo
4. Paste it into the skill editor → Save
5. Start using it in your next conversation
```

No setup. No API keys. No code. Copy, paste, go.

## 🧩 Skill Format

Every skill follows a consistent template ([`SKILL_TEMPLATE.md`](./template/SKILL_TEMPLATE.md)):

```
├── Frontmatter        → Name, description, trigger keywords
├── Summary             → One-sentence purpose
├── Input Definition    → What the user provides
├── Thinking Process    → Context analysis before output
├── Quality Criteria    → What "good" looks like
├── Guidelines          → Dimension-specific best practices
└── Anti-patterns       → What to avoid
```

This structure ensures every skill produces consistent, high-quality, context-aware output — not generic filler.

## 🤝 Contributing

Found a use case that's missing? Want to improve an existing skill? PRs are welcome.

**Before you submit:**
1. Follow the [`SKILL_TEMPLATE.md`](./template/SKILL_TEMPLATE.md) format
2. Test your skill in Claude with at least 3 different inputs
3. Include a real example (input → output) in your PR description
4. Place your skill in the correct role folder

Open an [issue](../../issues) to request a specific skill or suggest a new role category.
